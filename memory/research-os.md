# Research OS

## 核心设计原则

1. **研究状态外置** — 所有研究状态（问题、文献、笔记、证据）存储在本地 Markdown 文件中，这是唯一的真实状态源（single source of truth）
2. **模型无关** — 任何 AI 模型（Claude Code、飞书机器人、未来模型）都只是文件系统的读写接口，不拥有任何真实状态。模型可随时更换，状态不依赖模型上下文
3. **可持久、可迁移** — 任意新模型只需读取目录结构即可无缝接管，不需要"恢复对话"或"导入历史"
4. **Git 做版本同步** — 所有变更通过 Git 追踪，可回溯到任意历史状态
5. **先验证再自动化** — 在数据结构稳定前不做自动化工具，先用真实数据手动跑通最小闭环

## 人机分工

- **人**：研究方向判断、价值判断、最终决策（不可压缩部分）
- **AI**：检索、整理、追踪、结构化（可自动化的认知劳动）

## 演化策略

- 从现有系统长出来，以"文献-问题-笔记-证据"为锚点
- 先做最小闭环，做到极致（有状态、有历史、有可恢复性）
- 再逐步接入其他能力（检索、写作、提醒、任务图）

## 数据结构

### 目录结构
```
research-os/
├── questions/          ← 研究问题
├── literature/         ← 文献卡片
├── notes/              ← 研究笔记
├── evidence/           ← 证据条目
├── index.md            ← 全局索引（后期自动生成）
└── changelog.md        ← 变更历史
```

### 文件格式
- YAML 头信息（id、type、status、created、关联 ID、tags）
- Markdown 正文（人可直接阅读）
- ID 互相引用形成知识网络

### ID 编号规则
- Q001, Q002... — 问题
- L001, L002... — 文献
- N001, N002... — 笔记
- E001, E002... — 证据
